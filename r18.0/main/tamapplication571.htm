Category: (2) TAM Application Type

Application Identifier: 5.3.7

Maturity Level: 3

Overview

Customer Order Enrichment application enables acquiring the missing parts of the order information.

Functionality

Complete customer and order data is needed during the various stages of order handling. Main application function is to acquire missing order data from surrounding systems (often values taken from catalogs and inventories, billing, fraud, etc.), which sometimes include external 3rd party systems (like country's common address or credit check system). Enriching values can be calculated on-the-fly from existing data and ordering rules. Requiring enriched data in simple channels introduces complexity in them. Creating the application for enrichment adding more channels becomes easier. Customers and their orders are recognized by parsing the minimal order data that is acquired later on. Sometimes the order data is not available during order entry/capture. Enrichment process can be active during all phases of customer order processing and execution, either automatically (with order values becoming available after execution in service/resource layers) or manually (with data entered during normal execution or fault correction).
